# DKP Loot Assistant - План тестирования

## Подготовка к тестированию

### Проверка установки

```lua
-- 1. Проверить загрузку модуля
/la help

-- 2. Проверить наличие функций
/dump SOTA_LA_OpenWindow
/dump SOTA_LA_OnLootOpened
/dump SOTA_ScannedLoot

-- 3. Проверить UI фрейм
/dump getglobal("SOTA_LootAssistantFrame")

-- Всё должно возвращать function или table
```

---

## Тест 1: Сканирование лута (автоматическое)

### Сценарий
1. Стать Master Looter (в группе/рейде)
2. Убить моба/открыть сундук с эпиками
3. Открыть лут

### Ожидаемый результат
- Автоматически открывается окно Loot Assistant
- Список содержит отсканированные эпики
- У каждого предмета:
  - Иконка отображается
  - Название правильного цвета (фиолетовое для эпиков)
  - Статус "Ожидает разрола"
  - Кнопка [Roll] активна
  - Кнопка [Выдать] неактивна

### Проверка
```lua
-- Количество отсканированных предметов
/script local count = 0; for k,v in pairs(SOTA_ScannedLoot) do count = count + 1 end; DEFAULT_CHAT_FRAME:AddMessage("Items: "..count)

-- Проверить данные первого предмета
/script for k,v in pairs(SOTA_ScannedLoot) do DEFAULT_CHAT_FRAME:AddMessage(v.name.." - "..v.status) break end
```

## Тест 3: Запуск одного аукциона

### Сценарий
1. В окне Loot Assistant нажать [Roll] у любого предмета
2. Дождаться открытия окна SOTA Auction

### Ожидаемый результат
- Открывается стандартное окно SOTA с аукционом
- Предмет в окне совпадает с выбранным
- Статус предмета в Loot Assistant меняется на "Аукцион идёт..."
- Таймер отсчитывается

### Проверка
```lua
-- Статус аукциона SOTA
/dump SOTA_GetAuctionState()  -- должно быть 10 (STATE_AUCTION_RUNNING)

-- Текущий предмет
/dump AuctionedItemLink

-- Статус в Loot Assistant
/script for k,v in pairs(SOTA_ScannedLoot) do if v.status == "rolling" then DEFAULT_CHAT_FRAME:AddMessage("Rolling: "..v.name) end end
```

## Тест 4: Принятие ставки и назначение победителя

### Сценарий
1. Аукцион запущен (из Теста 3)
2. Игрок делает ставку через whisper: `/w YourName bid 500`
3. В окне SOTA нажать [Accept] на ставке
4. Нажать [Award] в SOTA

### Ожидаемый результат
- Статус предмета в Loot Assistant меняется на "Победитель: PlayerName"
- Кнопка [Roll] становится неактивной
- Кнопка [Выдать] становится активной
- В чате появляется: `[Loot Assistant] Победитель: PlayerName (500 ДКП)`

### Проверка
```lua
-- Проверить победителя
/script for k,v in pairs(SOTA_ScannedLoot) do if v.winner then DEFAULT_CHAT_FRAME:AddMessage("Winner: "..v.winner.." for "..v.name) end end

-- Статус
/script for k,v in pairs(SOTA_ScannedLoot) do DEFAULT_CHAT_FRAME:AddMessage(v.name..": "..v.status) end
```

## Тест 5: Прямая выдача из окна лута (GiveMasterLoot)

### Сценарий
1. Предмет разыгран, победитель назначен (из Теста 4)
2. **Окно лута ещё открыто** (важно!)
3. Победитель в рейде/группе
4. Нажать [Выдать] в Loot Assistant

### Ожидаемый результат
- Предмет автоматически выдаётся победителю
- Статус меняется на "Выдан: PlayerName"
- В рейд-чат: `Выдан [Item] → PlayerName`
- Кнопка [Выдать] становится неактивной

### Проверка
```lua
-- Проверить статус
/script for k,v in pairs(SOTA_ScannedLoot) do if v.status == "given" then DEFAULT_CHAT_FRAME:AddMessage("Given: "..v.name.." to "..v.winner) end end
```

### Если не работает
- Проверить, что окно лута открыто: `/dump GetNumLootItems()`
- Проверить, что игрок в рейде: `/dump GetNumRaidMembers()`
- Попробовать вручную: `/dump GiveMasterLoot(1, 2)` (слот 1, кандидат 2)

## Тест 6: Выдача через трейд (fallback)

### Сценарий
1. Предмет разыгран, победитель назначен
2. **Закрыть окно лута** (ESC или взять предмет в сумку)
3. Победитель открывает трейд с вами

### Ожидаемый результат
- Автоматически открывается окно Trade Helper
- Список предметов для этого игрока
- Кнопка [Выдать Всё]

### Действие
4. Нажать [Выдать Всё]

### Ожидаемый результат после нажатия
- Все предметы найдены в сумках
- Предметы помещены в трейд-окно
- Статус меняется на "Выдан"
- Сообщение: `[Loot Assistant] Помещено в трейд: X предметов`

### Проверка
```lua
-- Проверить, что предметы в сумках
/script for k,v in pairs(SOTA_ScannedLoot) do local bag, slot = SOTA_LA_FindItemInBags(v.itemId); DEFAULT_CHAT_FRAME:AddMessage(v.name.." in bag: "..(bag or "nil")) end
```

## Тест 7: Кнопка "Разыграть Всё"

### Сценарий
1. Создать 3+ тестовых предмета (см. Тест 2) или отсканировать лут
2. Все предметы в статусе "pending"
3. Нажать [Разыграть Всё]

### Ожидаемый результат
- Первый аукцион запускается сразу
- После завершения первого (через 2 сек) запускается второй
- И так далее для всех предметов
- Сообщение: `[Loot Assistant] Запуск аукционов для 3 предметов...`

### Проверка
```lua
-- Проверить очередь
/dump RollQueue  -- должна быть таблица с ключами предметов

-- Проверить таймер
/dump RollQueueTimer  -- счётчик до следующего аукциона
```

## Тест 8: Удаление предмета из списка

### Сценарий
1. В списке есть предмет
2. Нажать кнопку [X] у предмета

### Ожидаемый результат
- Предмет исчезает из списка
- Сообщение: `[Loot Assistant] Предмет удалён из списка`
- Предмет удаляется из `SOTA_ScannedLoot`

### Проверка
```lua
-- До удаления
/script local count = 0; for k,v in pairs(SOTA_ScannedLoot) do count = count + 1 end; DEFAULT_CHAT_FRAME:AddMessage("Before: "..count)

-- [Нажать X]

-- После удаления
/script local count = 0; for k,v in pairs(SOTA_ScannedLoot) do count = count + 1 end; DEFAULT_CHAT_FRAME:AddMessage("After: "..count)
```

## Тест 9: Обновление slotIndex после выдачи

### Сценарий
1. Открыть лут с 3+ предметами
2. Выдать предмет из слота 1
3. Проверить, что слоты обновились

### Ожидаемый результат
- После выдачи предмета из слота 1:
  - Слот 2 стал слотом 1
  - Слот 3 стал слотом 2
- Модуль автоматически обновляет `entry.slot` для всех предметов

### Проверка
```lua
-- До выдачи
/script for k,v in pairs(SOTA_ScannedLoot) do DEFAULT_CHAT_FRAME:AddMessage(v.name.." slot="..v.slot) end

-- [Выдать предмет]

-- После выдачи
/script for k,v in pairs(SOTA_ScannedLoot) do DEFAULT_CHAT_FRAME:AddMessage(v.name.." slot="..v.slot) end
```

## Тест 10: Фильтры качества

### Сценарий
1. Изменить минимальное качество:
```lua
/script SOTA_CONFIG_LA_MinQuality = 3  -- синие и выше
/reload
```
2. Открыть лут с синими и эпическими предметами

### Ожидаемый результат
- Сканируются синие (rare) и эпические (epic) предметы
- Зелёные (uncommon) игнорируются

### Проверка
```lua
-- Проверить качество всех отсканированных предметов
/script for k,v in pairs(SOTA_ScannedLoot) do DEFAULT_CHAT_FRAME:AddMessage(v.name.." quality="..v.quality) end

-- Все quality >= 3
```

## Тест 11: Исключение квестовых предметов

### Сценарий
1. Включить фильтр:
```lua
/script SOTA_CONFIG_LA_ExcludeQuest = 1
/reload
```
2. Открыть лут с квестовым эпиком

### Ожидаемый результат
- Квестовые предметы не попадают в список

### Альтернатива
```lua
/script SOTA_CONFIG_LA_ExcludeQuest = 0
/reload
```
- Квестовые предметы включаются в список

## Тест 13: Интеграция с коммуникацией (addon-канал)

### Сценарий
1. Запустить аукцион
2. Принять ставку победителя

## Известные баги (для проверки в будущем)

1. **Если закрыть окно SOTA во время аукциона**
   - Проверить, что статус в LA остаётся "rolling"
   - Должен быть способ сбросить статус

2. **Если игрок покинул рейд после победы**
   - GiveMasterLoot не сработает
   - Должно быть сообщение об ошибке

3. **Если несколько предметов с одинаковым itemId**
   - Проверить уникальность ключей (используется timestamp)
